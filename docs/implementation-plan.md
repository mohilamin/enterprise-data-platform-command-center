# Implementation Plan

## Phase 1: Foundation

Create repo structure, public README, AGENTS guidance, Python setup, Docker, CI, Makefile, and config files.

## Phase 2: Synthetic Platform Signals

Generate a data product catalog, platform health signals, SLA signals, lineage dependencies, usage signals, and cross-system incident scenarios.

## Phase 3: Normalized Control Plane

Normalize raw signals into a common data product health model, incident model, SLA model, governance risk model, AI readiness model, and lineage dependency model.

## Phase 4: Scoring and Correlation

Calculate data product health, platform health, AI readiness, governance risk, SLA compliance, and cross-system incident correlation.

## Phase 5: Outputs

Generate scorecards, executive summaries, lineage graphs, a DuckDB warehouse, API endpoints, dashboard sections, and tests.
